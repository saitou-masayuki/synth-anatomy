# シンセ解剖図（Synth Anatomy）

**公開URL: https://synth-anatomy.pages.dev/**

**信号の流れが見えるブラウザシンセ。** ソフトウェア・シンセサイザー（Serum / Vital などのウェーブテーブル系）の「ここをいじると、ここがこう変化する」というパラメーター間の繋がりを、音を鳴らしながら体感して学ぶための学習ツールです。

## 特徴

- **解剖図UI**: OSC → FILTER → AMP → OUT の信号フロー図そのものが操作パネル。各ブロックの出口で波形とスペクトルがリアルタイムに変化して見える
- **音と表示が同期**: LFO・エンベロープの「揺れ」は、音のスケジュールと画面の脈動が同じ数式（mod-engine.js）から計算される。出力遅延（outputLatency）も補正済み
- **今なにが起きた？パネル**: ノブを回すたびに「何をしたか・音がどうなるか・どこを見るか」を日本語で説明
- **モッドリング**: LFOを割り当てたノブの外周に揺れの範囲と現在位置を表示（Serum / Vital と同じ記法）
- **ウェーブテーブル体験**: WT位置ノブで sine→三角→ノコギリ→矩形 を連続モーフ。「フィルターで削る」と「波形自体が変わる」の違いを体感できる
- **PWA対応**: ビルド不要のバニラJavaScript。オフラインでも動作し、スマートフォンのホーム画面にも追加可能

## 使い方

3つのモードをヘッダーのタブで切り替えます。

- **さわる**: 自由に音作りをするサンドボックス。鍵盤（またはPCキー A W S E D F T G Y H U J K、Z/Xでオクターブ移動）で鳴らしながらノブを回す（Shift+ドラッグで微調整、ダブルクリックで初期値）。LFO1の「ノブにつなぐ」で、光ったノブをクリックすると配線される
- **つくる（音作りテスト）**: お手本の音を聴いて、手順どおりにノブを動かして再現する。プラック、ウォブルベース、WTモーション・パッドなど6本。できた音はそのまま使える
- **きく（聞き取りテスト）**: AとBの音を聴き比べて、どのノブが変わったかを当てる。全3レベル。間違えやすいパラメーターほど出題されやすくなる

「かんたん表示/くわしい表示」で表示するパラメーターの量を切り替えられます。

## 開発

```bash
# ローカルサーバー
python3 -m http.server 8767

# テスト（純粋ロジック層）
node --test test/
```

### アーキテクチャ

全機能（エンジン・UI・解説・レシピ・クイズ）が `content-params.js` の統一パラメーター定義を参照する。詳細は [docs/design.md](docs/design.md) を参照。

| ファイル | 役割 |
|---|---|
| `content-params.js` | 統一パラメーター定義＋解説（純粋データ） |
| `wavetables.js` | 教材ウェーブテーブルの倍音スペクトル定義 |
| `mod-engine.js` | LFO/ADSR数式・変調ルート解決（音と可視化の共有数式） |
| `describe-engine.js` | 「今なにが起きた？」の文生成 |
| `synth-engine.js` | Web Audioノードグラフ（テスト済み純粋コアの薄いラッパー） |
| `viz.js` | Canvasスコープ・SVGモジュレーション線（単一rAFループ） |
| `app.js` | UI結線・鍵盤・プリセット・iOS音声解錠 |
